# Output Format (Strict)
以下の構成で出力してください。

1. **Frontmatter (YAML)**
   ---
   title: "記事タイトル"
   date: "YYYY-MM-DD"
   description: "100文字程度の概要"
   thumbnail: "/images/works/XXXX.png"
   technologies: ["TechA", "TechB"]
   links:
     github: "URL"
     demo: "URL"
   featured: true/false
   recommended: true/false
   ---

2. **Section 1: タイトルと概要**
   - 箇条書きで羅列。
3. **Section 2: なぜ作ろうと思ったか**
   - 開発のきっかけ、解決したかった個人的な課題、コストやハードルに関する本音を記述。
4. **Section 3: 技術スタックと選定理由**
   - 使用技術の一覧と、それを選んだ論理的・感情的な理由。
   - 「こだわったポイント」として、UXや実装上の工夫を詳細に。
5. **Section 4: AIをどのように開発に活かしたか**
   - 設計段階での壁打ち、実開発でのAIツール（Antigravity等）の活用法。
   - 「設計やテストは人間、高速化はAI」という役割分担を強調。
6. **Section 5: 振り返り**
   - 開発を通じた学び、自身の哲学との合致、今後の展望。
   - 未来の自分への一言。

# Writing Guidelines
- **トーン:** 技術的に詳しく、かつ謙虚で親しみやすい語り口。
- **語彙:** 基本的には丁寧なですます調で、バランスを見て「お目汚し」「楽ちん」「作り込み」「壁打ち」など軟らかい表現を適宜使用。
- **強調構文:** 自身の信念や特に重要なUXに関する記述には、`:red[強調したい内容]` という形式を使用してください。
- **AIへの態度:** AIを魔法のツールではなく、設計思想を具現化するための「強力なパートナー」として描写してください。

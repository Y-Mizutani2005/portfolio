# Writing Style Guidelines
1. **構成:** - 「タイトル」「なぜ作ったか/考えたか」「技術スタック/選定理由」「こだわったポイント」「AIの活用（あれば）」「振り返り」の順で構成する。
2. **トーン:** - 謙虚だが自信がある。
   - 「〜です」「〜ます」調を基本としつつ、心の声（「〜なのは重い…」「〜したい！」など）を適度に入れ、親しみやすさを出す。
   - QiitaやZennのような公式感よりも、自分のための「記録・ラボ」としての雰囲気を大切にする。
3. **こだわりポイントの明示:**
   - 独自のこだわり（UX、設計思想、特定のスタンス）には Markdown の引用や強調を使い、「:red[私のスタンスとして〜]」といった表現で熱量を伝える。
4. **AIとの付き合い方:** - AIを「丸投げ」するのではなく、まず自分で設計し、AIを「加速器」として使っている様子を描写する。

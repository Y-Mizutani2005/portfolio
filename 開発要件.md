# 個人ブログ兼ポートフォリオサイト 要件定義書

**作成日**: 2025年10月26日  
**バージョン**: 1.0

---

## 1. プロジェクト概要

### 1.1 目的・背景
- 学習のアウトプットとして発信するモチベーションを維持する
- 就職活動に向けて、技術力やポートフォリオをアピールできるWebサイトを構築する
- 技術的な内容だけでなく、趣味（ゲーム、グルメ）に関する発信も行う

### 1.2 ターゲットユーザー
- **エンジニア仲間**: 技術記事を通じた知識共有・交流
- **採用担当者**: 就職活動における技術力・ポートフォリオの確認
- **ネット上の趣味人**: ゲーム・グルメなどの趣味コンテンツの閲覧者

### 1.3 プロジェクトのゴール
- 技術ブログとポートフォリオを統合した個人サイトの構築
- GitHubへのプッシュで自動的にコンテンツが更新される仕組みの実現
- SEOに強く、モダンなユーザー体験を提供するサイトの実現

## 2. 機能要件

### 2.1 ページ構成

#### 2.1.1 トップページ (`/`)
- 自己紹介セクション
- スキル・技術スタック表示
- 注目プロジェクト（3〜5件）
- 最新ブログ記事（3〜5件）
- SNS・GitHubへのリンク

#### 2.1.2 ポートフォリオページ (`/works`)
- 制作物の一覧表示（カード形式）
- 各プロジェクトの情報
  - タイトル
  - サムネイル画像
  - 概要
  - 使用技術
  - GitHubリポジトリ・デモサイトへのリンク
- フィルタリング機能（技術スタック別）

#### 2.1.3 ブログ一覧ページ (`/blog`)
- 記事の一覧表示（カード形式）
- 各記事カードの情報
  - タイトル
  - 投稿日
  - カテゴリ
  - タグ
  - 概要（excerpt）
  - サムネイル画像（オプション）
- カテゴリフィルター（Tech / Life）
- タグによる絞り込み機能
- ページネーションまたは無限スクロール

#### 2.1.4 ブログ記事詳細ページ (`/blog/[slug]`)
- 記事本文（Markdown レンダリング）
- 記事メタ情報（投稿日、更新日、カテゴリ、タグ）
- 目次（Table of Contents）自動生成
- シンタックスハイライト対応
- LaTeX 数式表示対応
- 前の記事・次の記事へのナビゲーション
- SNSシェアボタン

#### 2.1.5 プロフィールページ (`/about`)
- 詳細な自己紹介
- 経歴（学歴・職歴）
- スキルセット詳細
- 資格・受賞歴
- 趣味・興味
- お問い合わせ方法

### 2.2 共通機能

#### 2.2.1 ヘッダー
- サイトロゴ / タイトル
- ナビゲーションメニュー（Home / Works / Blog / About）
- ダークモード切り替えボタン

#### 2.2.2 フッター
- コピーライト
- SNSリンク
- プライバシーポリシー・免責事項へのリンク（オプション）

#### 2.2.3 ダークモード
- ライト/ダークテーマの切り替え機能
- ユーザーの選択を記憶（localStorage）
- システム設定に追従するオプション

#### 2.2.4 レスポンシブ対応
- モバイル（〜640px）
- タブレット（641px〜1024px）
- デスクトップ（1025px〜）

### 2.3 コンテンツ管理

#### 2.3.1 ブログ記事
- Markdownファイルで管理
- frontmatter によるメタデータ管理
  ```yaml
  ---
  title: "記事タイトル"
  date: "2025-10-26"
  updated: "2025-10-27" # オプション
  category: "Tech" # Tech | Life
  tags: ["Next.js", "TypeScript", "React"]
  excerpt: "記事の概要文"
  thumbnail: "/images/blog/sample.jpg" # オプション
  draft: false # 下書き機能
  ---
  ```

#### 2.3.2 ポートフォリオ作品
- Markdownまたは JSON ファイルで管理
- 必要な情報
  ```yaml
  ---
  title: "プロジェクト名"
  date: "2025-10-26"
  description: "プロジェクトの説明"
  thumbnail: "/images/works/sample.jpg"
  technologies: ["Next.js", "TypeScript", "Tailwind CSS"]
  links:
    github: "https://github.com/..."
    demo: "https://demo.example.com"
  featured: true # トップページに表示するか
  ---
  ```

#### 2.3.3 画像管理
- `/public/images/` ディレクトリで管理
  - `/public/images/blog/` - ブログ記事用
  - `/public/images/works/` - ポートフォリオ用
  - `/public/images/common/` - 共通画像
- Next.js の `<Image>` コンポーネントで最適化

## 3. 非機能要件

### 3.1 パフォーマンス
- Lighthouse スコア 90点以上（全項目）
- First Contentful Paint (FCP) < 1.8秒
- Largest Contentful Paint (LCP) < 2.5秒
- 画像の遅延読み込み（lazy loading）
- コード分割による最適化

### 3.2 SEO
- **メタタグ最適化**
  - タイトル、ディスクリプション
  - OGP (Open Graph Protocol)
  - Twitter Card
- **sitemap.xml 自動生成**
- **robots.txt 設定**
- **構造化データ (JSON-LD)**
  - Article（ブログ記事）
  - Person（プロフィール）
  - WebSite
- **セマンティックHTML**
  - 適切な見出しタグ（h1, h2, h3...）
  - article, section, nav などの使用

### 3.3 アクセシビリティ
- WCAG 2.1 AA レベル準拠を目指す
- キーボードナビゲーション対応
- 適切な alt テキスト
- カラーコントラスト比の確保

### 3.4 セキュリティ
- HTTPS 通信（Vercel デフォルト）
- Content Security Policy (CSP) 設定
- XSS 対策（React のデフォルト機能）

### 3.5 アナリティクス
- Vercel Analytics の導入
- ページビュー、ユニークビジター数の計測
- （将来的に Google Analytics 4 の追加を検討）

## 4. 技術仕様

### 4.1 技術スタック

#### フロントエンド
- **フレームワーク**: Next.js 15 (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **UI ライブラリ**: Radix UI / Headless UI（必要に応じて）

#### Markdown処理
- **パーサー**: unified / remark / rehype
- **シンタックスハイライト**: shiki
- **数式表示**: KaTeX
- **目次生成**: remark-toc または自作

#### その他ライブラリ
- **ダークモード**: next-themes
- **アナリティクス**: @vercel/analytics
- **日付処理**: date-fns
- **アイコン**: lucide-react または react-icons

#### 開発ツール
- **パッケージマネージャー**: npm / yarn / pnpm
- **Linter**: ESLint
- **Formatter**: Prettier
- **Git**: GitHub

#### ホスティング・デプロイ
- **ホスティング**: Vercel
- **デプロイ**: GitHub連携による自動デプロイ
- **ドメイン**: Vercelのデフォルトドメイン or カスタムドメイン

## 5. デザイン仕様

### 5.1 カラーパレット

#### ライトモード
- **Primary**: 紫系（#8B5CF6 - Violet 500）
- **Secondary**: 紫系の薄い色（#A78BFA - Violet 400）
- **Background**: 白〜薄いグレー（#FFFFFF, #F9FAFB）
- **Text**: ダークグレー（#111827 - Gray 900）
- **Text Secondary**: グレー（#6B7280 - Gray 500）
- **Border**: ライトグレー（#E5E7EB - Gray 200）

#### ダークモード
- **Primary**: 紫系（#A78BFA - Violet 400）
- **Secondary**: 紫系の濃い色（#8B5CF6 - Violet 500）
- **Background**: 黒〜ダークグレー（#0F0F0F, #1A1A1A）
- **Text**: 白〜ライトグレー（#F9FAFB - Gray 50）
- **Text Secondary**: グレー（#9CA3AF - Gray 400）
- **Border**: ダークグレー（#374151 - Gray 700）

### 5.2 タイポグラフィ

#### フォントファミリー
- **本文**: system-ui, -apple-system, sans-serif
- **コード**: 'Fira Code', 'JetBrains Mono', monospace
- **見出し**: 本文と同じ（太さで差別化）

#### フォントサイズ（Tailwind基準）
- **H1**: text-4xl (36px) / md:text-5xl (48px)
- **H2**: text-3xl (30px) / md:text-4xl (36px)
- **H3**: text-2xl (24px) / md:text-3xl (30px)
- **H4**: text-xl (20px) / md:text-2xl (24px)
- **Body**: text-base (16px)
- **Small**: text-sm (14px)

### 5.3 スペーシング
- セクション間: py-16 (64px) / md:py-24 (96px)
- 要素間: space-y-4 (16px) / space-y-6 (24px)
- コンテナ最大幅: max-w-7xl (1280px) / ブログ記事: max-w-3xl (768px)

### 5.4 主要コンポーネントのデザイン

#### ブログカード
- 角丸: rounded-lg
- シャドウ: shadow-md (hover: shadow-lg)
- トランジション: transition-all duration-300
- サムネイル: aspect-video

#### ボタン
- Primary: 紫背景 + 白文字
- Ghost: 透明背景 + 紫文字
- ホバーエフェクト: 明度変化 + スケール

#### コードブロック
- 背景: ダークグレー（両モード共通）
- 角丸: rounded-md
- パディング: p-4
- 行番号表示（オプション）
- コピーボタン

## 6. 開発・運用

### 6.1 開発フロー
1. ローカル環境でNext.jsプロジェクトをセットアップ
2. GitHubリポジトリを作成
3. Vercelと連携
4. 機能ごとにブランチを切って開発
5. プルリクエスト → レビュー → マージ
6. main ブランチへのマージで自動デプロイ

### 6.2 ブログ記事公開フロー
1. `/content/blog/` にMarkdownファイルを作成
2. frontmatter とコンテンツを記述
3. ローカルで確認（`npm run dev`）
4. GitHubにプッシュ
5. Vercelが自動でビルド・デプロイ
6. 本番環境に反映

### 6.3 バージョン管理
- **メインブランチ**: `main`（本番環境）
- **開発ブランチ**: `develop`（オプション）
- **機能ブランチ**: `feature/xxx`, `fix/xxx`

## 7. 制約事項・前提条件

### 7.1 制約事項
- Vercel の無料プランを使用（月100GB帯域、100GBストレージ）
- コメント機能は実装しない
- RSS フィードは実装しない
- 初期段階では画像はGitHubで管理（将来的にCDN移行を検討）

### 7.2 前提条件
- Next.js 初学者として、学びながら開発を進める
- 大学生の就職活動用という目的を意識した設計
- 技術記事と趣味コンテンツの両立

## 8. 成功指標（KPI）

### 8.1 技術的指標
- Lighthouse スコア: 全項目 90点以上
- ページ読み込み速度: LCP < 2.5秒
- ビルドエラーゼロ

### 8.2 コンテンツ指標
- 月間ブログ投稿: 2〜4記事
- ポートフォリオ作品: 初期5件以上

### 8.3 アクセス指標（参考）
- 月間ユニークビジター（目標は運用開始後に設定）
- 平均セッション時間
- 直帰率
